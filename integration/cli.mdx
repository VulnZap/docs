## Installation

Install VulnZap globally:

```bash
npm install -g vulnzap
```

## Quick Start

### 1. Complete Setup (Recommended)

Complete onboarding with API key setup and IDE integration:

```bash
npx vulnzap init
```

**This command will:**
- Guide you through API key setup
- Automatically detect installed IDEs (VS Code, Cursor, Windsurf)
- Allow you to select multiple IDEs for integration
- Configure MCP settings and install extensions
- Set up your development environment for secure coding

### 2. Manual Setup (Alternative)

```bash
# Setup API key only
vulnzap setup -k <your-api-key>

# Setup API key with specific IDE (cursor|windsurf|cline|vscode)
vulnzap setup -k <your-api-key> --ide cursor
```

## Commands Reference

### Setup & Configuration

#### `vulnzap init`

Complete setup with guided onboarding (recommended):

```bash
vulnzap init
```

Interactive wizard that handles API key setup and IDE integration.

#### `vulnzap setup`

Setup API key manually:

```bash
# Setup API key only
vulnzap setup -k <api-key>

# Setup API key with specific IDE
vulnzap setup -k <api-key> --ide <cursor|vscode|windsurf>
```

**Options:**
- `-k, --key <api-key>` - Your VulnZap API key
- `--ide <ide>` - Specific IDE to configure (cursor, vscode, windsurf, cline)

#### `vulnzap account`

Check account information:

```bash
vulnzap account
```

Displays your account details, plan information, and usage statistics.

#### `vulnzap status`

Check server status:

```bash
vulnzap status
```

Shows connection status and MCP server health.

#### `vulnzap help`

View help information:

```bash
vulnzap help
```

### Security Scanning

#### `vulnzap check`

Check individual package for vulnerabilities:

```bash
# Standard format
vulnzap check <ecosystem:package@version>

# Examples
vulnzap check npm:express@4.17.1
vulnzap check pip:requests@2.25.1
vulnzap check pypi:django@3.1.0

# Alternative syntax
vulnzap check <package> --ecosystem <eco> --version <ver>
vulnzap check express --ecosystem npm --version 4.17.1
```

**Supported Ecosystems:**
- `npm` - Node.js packages
- `pip` / `pypi` - Python packages
- `maven` - Java packages
- `go` - Go modules
- `cargo` - Rust crates
- `composer` - PHP packages
- `rubygems` - Ruby gems

#### `vulnzap batch-scan`

Batch scan packages in current directory:

```bash
# Scan all package files in current directory
vulnzap batch-scan

# Scan specific ecosystem
vulnzap batch-scan --ecosystem npm

# Output to file
vulnzap batch-scan --output scan-results.json
```

**Options:**
- `--ecosystem <eco>` - Limit to specific package ecosystem
- `--output <file>` - Save results to file

**Detects:**
- `package.json` (npm)
- `requirements.txt` (pip)
- `pom.xml` (maven)
- `go.mod` (go)
- `Cargo.toml` (cargo)
- `composer.json` (php)
- `Gemfile` (ruby)

#### `vulnzap scan`

Repository scan (GitHub):

```bash
vulnzap scan https://github.com/owner/repo

# With options
vulnzap scan https://github.com/owner/repo \
  --branch main \
  --wait \
  --output report.json \
  --key your_api_key
```

**Options:**
- `--branch <branch>` - Specific branch to scan (default: main)
- `--wait` - Wait for scan completion
- `--output <file>` - Save results to file
- `--key <api-key>` - Override API key for this scan

### IDE Integration

#### `vulnzap connect`

Connect to a specific IDE (alternative to init/setup):

```bash
# Interactive IDE selection
vulnzap connect

# Connect to specific IDE
vulnzap connect --ide cursor
vulnzap connect --ide windsurf
vulnzap connect --ide cline
vulnzap connect --ide vscode
```

Configures MCP settings and installs necessary extensions.

#### `vulnzap mcp`

Start MCP server manually (for IDEs that use MCP):

```bash
vulnzap mcp
```

Runs the Model Context Protocol server for IDE integration. Typically called automatically by IDE configuration files.
