## Overview

VulnZap provides comprehensive team management features for organizations to collaborate on security, manage access, and maintain audit trails.

<Info>
  Team features are available on **Standard** plans and above.
</Info>

## Team Members

### Inviting Team Members

<Steps>
  <Step title="Navigate to Team Settings">
    Go to [Dashboard](https://vulnzap.com/dashboard) â†’ **Settings** â†’ **Team**
  </Step>
  
  <Step title="Click Invite Member">
    Click **Invite Team Member** button
  </Step>
  
  <Step title="Enter Details">
    - Email address
    - Role (Owner, Admin, Member, Read-Only)
    - Projects (optional: limit access to specific projects)
  </Step>
  
  <Step title="Send Invitation">
    Member receives email invitation with onboarding link
  </Step>
  
  <Step title="Member Accepts">
    They create account and join your team workspace

### Roles and Permissions

<Tabs>
  <Tab title="Owner">
    **Full Access**
    - All Admin permissions
    - Manage billing and subscriptions
    - Add/remove team members
    - Delete organization
    - Transfer ownership
    
    **Limitations:**
    - Only one owner per organization
    - Cannot be removed (must transfer ownership first)
  </Tab>
  
  <Tab title="Admin">
    **Administrative Access**
    - View all projects and findings
    - Run scans and apply fixes
    - Invite/remove team members (except Owner)
    - Manage API keys
    - Configure integrations
    - Access audit logs
    
    **Limitations:**
    - Cannot manage billing
    - Cannot delete organization
  </Tab>
  
  <Tab title="Member">
    **Standard Access**
    - View assigned projects
    - Run scans
    - Apply fixes (with approval)
    - Create API keys (for own use)
    - View scan history
    
    **Limitations:**
    - Cannot invite members
    - Cannot access all projects
    - Cannot modify team settings
  </Tab>
  
  <Tab title="Read-Only">
    **View-Only Access**
    - View assigned projects
    - View findings and reports
    - Export scan results
    - View dashboard metrics
    
    **Limitations:**
    - Cannot run scans
    - Cannot apply fixes
    - Cannot create API keys
    - Cannot modify anything
  </Tab>
</Tabs>

## Project Access Control

### Assigning Members to Projects

Limit member access to specific projects:

<Steps>
  <Step title="Go to Project Settings">
    Dashboard â†’ **Projects** â†’ Select project â†’ **Settings**
  </Step>
  
  <Step title="Access Control Tab">
    Navigate to **Access Control** tab
  </Step>
  
  <Step title="Add Members">
    - Click **Add Member**
    - Select team member(s)
    - Choose permission level (Read, Write, Admin)
  </Step>
  
  <Step title="Save Changes">
    Member now has access to this project only
  </Step>
</Steps>

### Project Permission Levels

**Read:**
- View findings
- View scan history
- Export reports

**Write:**
- Read permissions +
- Run scans
- Apply fixes
- Comment on findings

**Admin:**
- Write permissions +
- Project settings
- Manage project members
- Delete project

## API Key Management

### Personal API Keys

Each team member can create personal API keys:

```bash
# Member creates personal key
vulnzap auth
```

**Scope:** Limited to projects the member has access to.

**Usage Limits:** Based on team's plan limits (shared quota).

### Shared API Keys

Admins can create shared team API keys:

<Steps>
  <Step title="Navigate to API Keys">
    Dashboard â†’ **Settings** â†’ **API Keys**
  </Step>
  
  <Step title="Create Team Key">
    Click **Create Team API Key**
  </Step>
  
  <Step title="Configure">
    - **Name**: "CI/CD Pipeline"
    - **Scope**: All projects or specific projects
    - **Permissions**: Read-only or Full access
    - **Expiration**: 90 days, 1 year, or Never
  </Step>
  
  <Step title="Copy Key">
    âš ï¸ **Important**: Copy immediately, won't be shown again
  </Step>
</Steps>

**Best Practices:**
- Use shared keys for CI/CD pipelines
- Use personal keys for developer workstations
- Rotate keys every 90 days
- Revoke keys when members leave

### Exporting Audit Logs

For compliance requirements:

```bash
# Export last 90 days
GET /api/v1/audit-logs?since=90d&format=csv

# Export specific date range
GET /api/v1/audit-logs?from=2025-01-01&to=2025-03-31&format=json
```

**Available Formats:** CSV, JSON, PDF

<Info>
  Audit log retention period depends on your plan: 7 days (Free), 30 days (Standard), 90 days (Scale), 365 days (Enterprise).
</Info>

Finding Details â†’ Assign To â†’ Select Member

### Fix Approval Workflow

For critical changes, require approval:

**Settings â†’ Team â†’ Approvals:**

```yaml
approval_required:
  - severity: critical
    approvers: 1  # Require 1 approval
    auto_merge: false
  
  - severity: high
    approvers: 0  # No approval needed
    auto_merge: true
```

**Workflow:**

1. Member applies fix to critical finding
2. Fix marked as "Pending Approval"
3. Admin/Owner reviews and approves
4. Fix is applied to codebase

## Seat Management

### Adding Seats

As your team grows:

<Steps>
  <Step title="Go to Billing">
    Dashboard â†’ **Settings** â†’ **Billing**
  </Step>
  
  <Step title="Manage Seats">

  </Step>

  </Step>
  
  <Step title="Confirm">

  </Step>

### Removing Seats

When members leave:

1. Remove member from team
2. Seat becomes available for new member
3. At end of billing period, downgrade unused seats

**No immediate charge/refund** - billing adjusts next cycle.

## Best Practices

<AccordionGroup>
  <Accordion title="Principle of Least Privilege">
    - Give members minimum necessary access
    - Use Read-Only role for stakeholders
    - Limit Admin role to 2-3 people
    - Regular access reviews quarterly
  </Accordion>
  
  <Accordion title="API Key Hygiene">
    - Rotate keys every 90 days
    - Use shared keys for CI/CD only
    - Revoke keys immediately when members leave
    - Monitor key usage in audit logs
  </Accordion>
  
  <Accordion title="Project Organization">
    - Group related projects
    - Assign clear project owners
    - Document project security requirements
    - Regular project health reviews
  </Accordion>
  
  <Accordion title="Communication">
    - Use comments for async discussions
    - @mention relevant team members
    - Set up Slack integration for real-time alerts
    - Weekly security standup meetings
  </Accordion>
</AccordionGroup>
