## Model Context Protocol (MCP)

VulnZap's IDE integration uses MCP to:
- Intercept AI-generated code before application
- Analyze code changes in real-time
- Provide inline security annotations
- Suggest secure alternatives to AI assistants

### How It Works

<Steps>
  <Step title="Code Generation">
    User prompts AI assistant or edits code manually
  </Step>
  
  <Step title="MCP Intercept">
    VulnZap receives the code diff via MCP
  </Step>
  
  <Step title="Real-Time Analysis">
    Scanning engine analyzes for vulnerabilities (<100ms)
  </Step>
  
  <Step title="Inline Feedback">
    Issues highlighted directly in editor
  </Step>
  
  <Step title="AI Enhancement">
    For AI-generated code, VulnZap suggests secure alternatives
  </Step>
  
  <Step title="Secure Application">
    Fixed code applied automatically or with user confirmation
  </Step>
</Steps>

  <Step title="Reload VS Code">

  </Step>

### Real-Time Inline Scanning

Code is scanned as you type or save:


**Inline Actions:**

## Configuration

### Per-Workspace Settings

Create `.vscode/vulnzap.json`:

```json
{
  "enabled": true,
  "scanOnSave": true,
  "scanOnType": false,
  "minSeverity": "high",
  "autoFix": false,
  "showInlineErrors": true,
  "showStatusBar": true,
  "excludePatterns": [
    "**/*.test.ts",
    "**/node_modules/**",
    "**/dist/**"
  ],
  "notifications": {
    "critical": true,
    "high": true,
    "medium": false,
    "low": false
  }
}
```

## Offline Mode

VulnZap can run offline with local vulnerability database:

```bash
# Download offline database
vulnzap db download

# Enable offline mode
vulnzap config set offline true
```

<Info>
  Offline mode has slightly reduced detection accuracy but provides basic protection without internet.
</Info>

## Performance

IDE integration is designed for minimal performance impact:

| Operation | Latency | Notes |
|-----------|---------|-------|
| Inline scan (on type) | <50ms | Debounced 300ms |
| File scan (on save) | <100ms | Full file analysis |
| AI code intercept | <100ms | Before code application |
| Package check | <200ms | Cached lookups |

**Performance Settings:**

```json
{
  "vulnzap.performance": {
    "debounceMs": 300,
    "maxFileSizeKB": 500,
    "enableCaching": true,
    "parallel": true
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="VulnZap Not Showing in IDE">
    **Solutions:**
    1. Verify MCP server is running: `vulnzap mcp status`
    2. Check API key is valid: `vulnzap auth verify`
    3. Restart IDE completely
    4. Check IDE logs for MCP errors
    5. Reinstall: `npm uninstall -g vulnzap && npm install -g vulnzap`
  </Accordion>
  
  <Accordion title="Scans Not Running">
    **Check:**
    - Is `scanOnSave` enabled in settings?
    - Is the file type supported (JS/TS/Python)?
    - Is the file excluded in `.vulnzapignore`?
    - Check status bar for error messages
    - Run manual scan: `Cmd/Ctrl+Shift+V S`
  </Accordion>
  
  <Accordion title="High CPU Usage">
    **Solutions:**
    1. Disable `scanOnType`, use only `scanOnSave`
    2. Exclude large files: `maxFileSizeKB: 500`
    3. Exclude test files and generated code
    4. Reduce `maxThreads` in config
    5. Use incremental scanning
  </Accordion>
  
  <Accordion title="False Positives">
    **Actions:**
    1. Right-click â†’ **Ignore This Instance** (with reason)
    2. Create suppression comment:
       ```javascript
       // vulnzap-disable-next-line sql-injection
       const query = buildDynamicQuery();
       ```
    3. Report false positive in dashboard for ML improvement
  </Accordion>
</AccordionGroup>
