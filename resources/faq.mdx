## General

<AccordionGroup>
  <Accordion title="What is VulnZap?">
    VulnZap is a security layer for agentic coding that provides real-time vulnerability scanning for AI-generated and human-written code. It detects security issues, suggests fixes, and integrates with popular IDEs and CI/CD pipelines.

    - Real-time scanning as you code
    - AI-aware (understands agent-generated code)

## Security & Privacy

<AccordionGroup>
  <Accordion title="Does VulnZap store my source code?">
    **No, by default VulnZap never stores your source code.**
    
    **What we store:**
    - Metadata (file paths, line numbers, vulnerability types)

<AccordionGroup>
  <Accordion title="What is agent-aware scanning?">
    **Agent-aware scanning** means VulnZap understands when code is generated by AI assistants (Cursor, GitHub Copilot, etc.) and can:
    
    1. **Intercept** AI-generated code before it's applied
    2. **Analyze** for security vulnerabilities
    3. **Provide feedback** to the AI with secure alternatives
    4. **Guide** the AI to regenerate secure code
    
    **Example:**
    AI generates SQL injection â†’ VulnZap detects â†’ Suggests parameterized queries â†’ AI regenerates secure code â†’ Applied to editor
    
    **Requires:** MCP protocol integration (Cursor, Windsurf, Cline)

    **Yes.** VulnZap generates context-aware patches for most vulnerabilities.
    
    **Auto-Fix Capabilities:**
    - **SQL Injection**: Parameterized queries
    - **XSS**: HTML escaping, sanitization

    **Application Methods:**
    - **Interactive**: Review each fix before applying
    - **Automatic**: Apply all fixes at once (use with caution)
    - **Export**: Generate patches for manual review
    
    **Success Rate:** ~85% of findings have auto-applicable fixes
  </Accordion>

## Technical

<AccordionGroup>
  <Accordion title="What is the MCP protocol?">
    **MCP (Model Context Protocol)** is an open standard that enables AI assistants to communicate with external tools and receive context-aware feedback.
    
    **How VulnZap uses MCP:**
    1. AI assistant (Cursor, etc.) generates code
    2. MCP sends code to VulnZap before applying
    3. VulnZap analyzes for vulnerabilities
    4. MCP sends security feedback back to AI
    5. AI regenerates secure code
    6. Secure code applied to editor
    
    **Benefits:**
    - Real-time security without slowing development
    - AI learns from security feedback
    - Prevents insecure code before it's written
    
    See [MCP Protocol](/integration/mcp-protocol) for technical details.
  </Accordion>
  
  <Accordion title="How fast are scans?">
    **Performance Benchmarks:**
    
    - **Inline (single file)**: <100ms
    - **Small repo** (100 files, 10K LOC): 5-10 seconds
    - **Medium repo** (1,000 files, 100K LOC): 20-40 seconds
    - **Large repo** (10,000 files, 1M LOC): 2-5 minutes
    
    **Incremental scans (CI/CD):**
    - Typical PR (10-20 changed files): <10 seconds
    
    **Optimization Features:**
    - Caching (unchanged files skip analysis)
    - Incremental mode (only changed files)
    - Parallel scanning (multi-core)
    - AST reuse (syntax trees cached)
  </Accordion>
